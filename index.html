Weather Synth

By Julia Dudlak & Hannah Yanover

Welcome to the Weather Synth

The Weather Synth is a generative synthesizer that transforms real-time global weather data into music. This idea is called data sonification: instead of looking at a number like “wind speed: 14 meters per second,” you actually hear it.

When you search for any city, we fetch live temperature, wind speed, humidity, and cloud cover. Each of these values is routed into a different synthesis engine running in parallel.

How Weather Becomes Sound

The four weather parameters each drive a distinct layer of sound:

  • Temperature → Pitch: Hotter cities produce higher drones, while colder cities sit low and dark.
  • Wind Speed → Noise: Faster wind increases turbulence and raises the resonant frequency.
  • Humidity → Harmonics: Dry air creates simple tones, while humidity adds rich, layered overtones.
  • Cloud Cover → Rain: Clear skies are quiet, while overcast conditions generate dense rhythmic drops.

The overall sound design aims to balance realism and musicality. Rather than directly simulating environmental audio, we construct an abstracted soundscape that preserves the character of the weather while remaining harmonically coherent.The goal is that every city has a unique sonic fingerprint. You shouldn’t need to look at the screen to tell the difference between two cities. You should be able to hear it.

Synthesis Architecture & Web Audio

FM Drone Layer

The drone uses frequency modulation synthesis with a carrier and modulator oscillator. Temperature maps to pitch across a pentatonic scale, ensuring musical output. We map temperature onto a pentatonic scale rather than a linear frequency range to avoid dissonance and ensure that arbitrary weather changes still produce musically coherent output. We layer a sub-oscillator and a detuned fifth to create a full harmonic texture. Because weather data updates discretely, we use AudioParam automation (e.g., linearRampToValueAtTime and setTargetAtTime) to interpolate between values and avoid discontinuities or audible artifacts.

Wind Layer

Wind is generated using subtractive synthesis. White noise is passed through a bandpass filter, with wind speed controlling both frequency and resonance. A slow LFO modulates the filter to simulate natural gusts.

Pad Layer

The pad uses additive synthesis. Humidity controls the number of harmonic partials, ranging from sparse tones to dense textures. Each harmonic includes slight modulation, creating a shimmering, chorus-like effect.

Rain Layer

Rain is generated through a stochastic process using randomized timing. Each droplet is a short sine burst with a downward pitch envelope, and is spatialized using stereo panning for realism.

Reverb & Spatialization

We generate a procedural convolution reverb using an impulse response created in real time. Spatialization is handled with panning so each layer occupies a different position in the sound field.

Beyond the Syllabus

One major addition was integrating a live weather API, allowing real-world data to drive the sound automatically. We also implemented spatial audio and procedural reverb, which were not covered in class.

From the course material, we applied FM synthesis, subtractive synthesis, additive synthesis, and AudioParam automation and ensured all transitions are smooth and click-free.

Interface Design

The UI is modeled after the Apple Weather app, with gradient-based city cards, persistent favorites using localStorage, and a detailed view with synth controls.

The goal was to make the data feel familiar, so the sound feels surprising.
